Construcción y Desarrollo de Software

Términos que a menudo se utilizan indistintamente, pero con enfoques y alcances distintos

Definiciones de Desarrollo de Software

Proceso Integral

Proceso de creación de un programa de software, abarcando todas las etapas del ciclo de vida del desarrollo de sistemas.

Concepción y Mantenimiento

Proceso de concepción, diseño, programación y mantenimiento de aplicaciones y sistemas informáticos.

Soluciones a Medida

Creación de soluciones informáticas personalizadas que se adaptan específicamente a los requisitos de una empresa.

Traducción de Necesidades

Proceso donde las necesidades del usuario se traducen en requerimientos de software que se implementan en código.

Etapas del Proceso de Desarrollo

1
Definición Inicial

Establecer objetivos, alcance del proyecto, acuerdos básicos y presupuesto preliminar.

2
Análisis de Requerimientos

Recopilar, examinar y obtener todos los requisitos del cliente, identificando restricciones.

3
Diseño

Planteamiento de la solución informática: arquitectura, estructuras de datos, interfaces y componentes.

4
Codificación/Desarrollo

Escribir código según especificaciones del diseño, siguiendo buenas prácticas de programación.

5
Pruebas

Revisar el software para detectar errores o defectos, asegurando que cumpla los objetivos.

6
Implementación

Instalar el software en el entorno del usuario, capacitar usuarios y asegurar operatividad.

7
Mantenimiento/Evolución

Manejar problemas, implementar cambios, corregir errores y añadir nuevas funcionalidades.

Buenas Prácticas y Calidad

✓ Código Limpio

Debe ser legible y bien estructurado para facilitar su reutilización y depuración.

✓ Arquitectura

Indispensable para analizar, diseñar y definir el estilo arquitectónico que cubra necesidades funcionales y no funcionales.

✓ Patrones de Diseño

Soluciones probadas a problemas de diseño comunes, reutilizables en diferentes contextos.

✓ Programación Extrema (XP)

Enfatiza simplicidad, comunicación, retroalimentación, coraje y respeto. Incluye prácticas como planificación incremental, releases pequeños, diseño simple, TDD, refactorización, programación en pares, propiedad colectiva e integración continua.

Principios SOLID

Responsabilidad Única

Una clase debe tener una sola razón para cambiar.

Abierto/Cerrado

Las entidades de software deben estar abiertas para extensión pero cerradas para modificación.

Sustitución de Liskov

Los objetos de un programa deberían ser reemplazables por instancias de sus subtipos sin alterar el correcto funcionamiento del programa.

Segregación de Interfaz

Muchas interfaces específicas son mejores que una interfaz de propósito general.

Inversión de Dependencia

Depender de abstracciones, no de implementaciones concretas.

En Resumen

La construcción o desarrollo de software es un proceso integral que abarca desde la conceptualización y planificación hasta el mantenimiento continuo, requiriendo un enfoque disciplinado y la aplicación de buenas prácticas y principios de ingeniería para asegurar la calidad y adaptabilidad del producto final.
